## ☁️ Day 1: What Is Cloud Computing?

### 🎯 Objective
The objective of this lab is to understand the core fundamentals of cloud computing, create an AWS account via the Free Tier, secure the root user with Multi-Factor Authentication (MFA), navigate the AWS Management Console, and set up a zero-spend billing budget to prevent unexpected charges.

### 📝 Lab Steps Summary
- Successfully established a foundational AWS Free Tier account.
- Enforced strong security on the root profile via MFA setup.
- Navigated the AWS console and explored diverse core services.
- Implemented a zero-spend alarm for proactive cost governance.

### 🔬 Lab Task Results

#### Step 1: AWS Account Creation
Created a new AWS account on the Free Tier and logged into the AWS Console to begin hands-on learning.
![Step 1 - AWS Account Creation](./DAY%201/Step%201%20-%20Console%20Home%20Page.png)

#### Step 2: MFA Setup
Secured the AWS root account by enabling Multi-Factor Authentication (MFA) to ensure robust identity protection.
![Step 2 - MFA Setup](./DAY%201/Step%202%20-%20MFA%20Device.png)

#### Step 3: Console Exploration
Explored the AWS Management Console, navigating through service dashboards like EC2, and familiarizing myself with the interface.

#### Step 4: Billing Budget Setup

### 🔒 Security Implementation
- **MFA setup on root account:** Strengthened the primary security posture of the AWS environment by requiring a time-based one-time password (TOTP) device in addition to standard credentials.

### 🛠️ AWS Services Explored
- **EC2:** Navigated the Virtual Server dashboard.
- **S3:** Explored simple storage service buckets.
- **IAM:** Accessed identity management to securely configure MFA.
- **RDS:** Reviewed relational database services.
- **Lambda:** Checked out serverless computing functions.

### 💸 Billing & Cost Control
- **Zero spend budget setup:** Established a strict financial boundary using AWS Budgets, ensuring email notifications are dispatched if forecasted or actual costs exceed $0.00.

### 💡 Key Takeaways
- Gained a foundational understanding of how cloud computing utilizes on-demand digital resources.
- Became comfortable navigating the primary AWS interface and identifying core services.
- Realized the critical importance of implementing both security (MFA) and financial governance (billing alarms) on day one.
